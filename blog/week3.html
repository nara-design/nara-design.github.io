user testing

User testing was done on the mid-fi prototype above with three testers. The testers are aged 21, 22 and 27. All user testers own smartphones.

testing plan

what i learned

  • navigation is intuitive, users were able to move around the different pages easily without getting stuck
  • users liked the options for block view + list view since users sometimes want to be able to scroll through headlines quickly, and sometimes want to just mindlessly scroll and mainly look at the images
  • users instinctively swiped up and then tapped outside of pop up to get rid of it, and then noticed the X button
  • feels off to users that the search bar + news categories are floating without a background behind them
  • users liked the app’s purpose being focused on positivity

next steps

  • change method for closing pop up to swiping up + tapping outside of pop up
  • add background behind floating search bar + news categories

hi-fi prototype

changes made

Since moving from the midfi prototype to the first hifi prototype, the following things were adjusted:

  • footer + header no longer float; they now cover the full width of the screen and start from the very top/bottom of the screen/li>
  • background behind search bar + news categories
  • new loading screen
  • “working” slider (can be interacted with)
  • interactive keyboard popup
  • flows can now loop endlessly (users can log out via flow 3 and start at flow 1 again)
  • positive pop up can be closed by tapping rather than swiping or X button (with text to help user if they get stuck)
