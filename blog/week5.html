Thu • 23/03/23

problem statement

Through the SWOT analyses, I came to the conclusion that public parking was the most logical and realistic problem to focus on. And so the problem statement was defined:

Driving in Christchurch is challenging because it is difficult to find parking options in the city.

This was then reframed as a "how might we" question:

How might we create a digital solution that helps people find parking options in Christchurch?

My solution is to create website that allows users to find available parking options for their destinations, as well as view information regarding those options (ie. time limits, fees, peak hours, etc).

field review

PayMyPark App

Strengths

  • allows user to set timer (and receive a notification when it goes off) so they won't go over the parking time limit
  • can use app as a casual user
  • app records payment information which will notify parking wardens if they try to ticket your vehicle
  • can top up and pay for parking directly through the app

Weaknesses

  • poor UX/UI design; difficult to navigate
  • can't find parking within the app itself—it opens a tab on your default phone browser on the CCC car park map site page
  • unintuitive user flow: have to find area code through CCC page, then return to the app and input the area code to book a parking spot
  • only allows user to buy parking spots (doesn't help user find free parking)
  • clicking on some options in the navigation takes the user to their default browser on their phone and onto the PayMyPark website, or the CCC website

Parkable App

Strengths

  • user flow: register/login → app asks where user is going → input destination → shows paid parking options → select an option → shows information (location, available bays, parking open hours, pricing per hour) → user reserves bay
  • user has 30 minutes to cancel reservation before they are charged
  • clean UX/UI design
  • allows user to hide full/closed car parks
  • emails receipts automatically to user's inbox

Weaknesses

  • struggles to function if user doesn't want app to track their location
  • going from menu to a new page and then back doesn't go back to menu; this is unintuitive
  • doesn't show free parking options
  • has a website which has many pages but doesn't actually do anything (user has to download app to accomplish anything)

Maps.me Site/App

Strengths

  • has both search functionality and filter functionality
  • search/filter results and map both visible at the same time
  • clean UX/UI design that follows design conventions
  • accessible both as a website and an app, depending on user's preference

Weaknesses

  • map functionality is pretty much useless; doesn't ever actually show the map
  • unintuitive user flow: home page → maps by city → search for city → select city → click a filter option → results show attractions "nearby" city → clicking a search result takes user to a new page about that location instead of showing the location on the big map
  • map moves around by itself based on user interactions, which might be helpful except the map itself is never visible (only the pins are visible)
  • only lets you select one filter at a time
  • filter option show up at the top of the page overlapping with the main navigation; can't easily click them

HERE WeGo App

Strengths

  • has offline mode
  • simple structure; easy, intuitive navigation
  • option to change map view (default, satelite)
  • pretty strong branding—prominent colour theme, illustration style
  • shows routes and time estimations for different forms of travel (car, bus, bike, walking)

Weaknesses

  • limited additional information regarding locations (eg. parking availability)
  • no way to "exit" the map and enter another page, which makes the app feel like there are always too many steps required to get to a page—constantly have to tap, then swipe up, then swipe again to exit, but exiting doesn't actually fully exit
